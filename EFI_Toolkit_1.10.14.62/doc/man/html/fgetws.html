FGETWS(3)	       FreeBSD Library Functions Manual 	      FGETWS(3)

NAME

     fgetws, getws - get a line from a stream

SYNOPSIS

     #include <wchar.h>

     wchar_t *
     fgetws(wchar_t *str, int size, FILE *stream)

     wchar_t *
     getws(wchar_t *str)

DESCRIPTION

     The fgetws() function reads at most one less than the number of wide characters
     specified by size from the given stream and stores them in the wide character
     string str. Reading stops when a newline character is found, at end-of-file or
     error.  The newline, if any, is retained.	If any characters are read and
     there is no error, a L`\0' wide character is appended to end the string.

     The getws() function is equivalent to fgetws() with an infinite size and a
     stream of stdin, except that the newline character (if any) is not stored
     in the string.  It is the caller's responsibility to ensure that the in-
     put line, if any, is sufficiently short to fit in the string.

RETURN VALUES

     Upon successful completion, fgetws() and getws() return a pointer to the
     wide character string.  If end-of-file occurs before any wide characters are 
     read, they return NULL and the buffer contents is unchanged.  If an error occurs,
     they return NULL and the buffer contents is indeterminate.  The fgetws() and
     getws() functions do not distinguish between end-of-file and error, and
     callers must use feof(3) and ferror(3) to determine which occurred.

ERRORS

     [EBADF]  The given stream is not a readable stream.

     The function fgetws() may also fail and set errno for any of the errors
     specified for the routines fflush(3),  fstat(2),  read(2),  or malloc(3).


     The function getws() may also fail and set errno for any of the errors
     specified for the routine getwchar(3).

SEE ALSO

     feof(3),  ferror(3),  fgetln(3)

STANDARDS

     The functions fgetws() and getws() conform to ISO 9899: 1990 (``ISO C'').
